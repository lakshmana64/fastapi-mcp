# Security Policy

## Supported Versions

Security fixes are provided on the latest released minor version.

## Reporting a Vulnerability

Please do not open public GitHub issues for potential vulnerabilities.

Instead, report privately with:

- Vulnerability description and impact
- Steps to reproduce
- A minimal proof of concept (if available)
- Suggested mitigation (optional)

If GitHub Security Advisories are enabled for this repository, use private
advisory reporting. Otherwise, contact maintainers directly.

## Response Targets

- Initial triage response: within 3 business days
- Status update: within 7 business days
- Remediation timeline: depends on severity and complexity

## Disclosure

We follow coordinated disclosure:

1. Report received privately
2. Issue validated and fixed
3. Release patch
4. Publish advisory/changelog notes

Please allow maintainers reasonable time to investigate and patch before public
disclosure.
